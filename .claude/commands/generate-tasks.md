# Generate Tasks

Break the feature spec (from `docs/specs/` or from this conversation) into an ordered, actionable
task list and write it to `docs/tasks/<feature>-tasks.md`.

Each task entry must include:

- A short imperative title
- The file(s) to create or modify
- Any prerequisite tasks
- A definition of done (one sentence)

Group tasks into logical phases (e.g., Core model, UI, Tests). Keep tasks small enough to complete
and verify independently. Create `docs/tasks/` if it does not exist.
