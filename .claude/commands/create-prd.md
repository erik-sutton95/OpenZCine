# Create PRD

Generate a lightweight Product Requirements Document for the feature or project described in this
conversation, then write it to `docs/PRD.md`.

The PRD must include:

- **Overview** — one-paragraph summary of the problem and goal
- **Users and context** — who this is for and how they currently work
- **Requirements** — prioritised list (MoSCoW or numbered)
- **Out of scope** — explicit exclusions
- **Success criteria** — measurable outcomes
- **Open questions** — unresolved decisions

Create `docs/` if it does not exist. Overwrite `docs/PRD.md` if one already exists, preserving any
locked sections marked `<!-- LOCKED -->`.
